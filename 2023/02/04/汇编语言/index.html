汇编语言学习笔记

Posted on 2023-02-04 | In 汇编语言 | 本文总阅读量 次

参考用书：《x86汇编语言：从实模式到保护模式》

第一部分 预备知识

预备知识部分主要包括：进制换算；处理器、内存和指令；汇编软件nasm等。

处理器

  1. 处理器的功能：自动化的取值执行

    处理器是一台电子计算机的核心，它会在振荡器脉冲的激励下，从内存中获取指令，并发起一些列由该指令所定义的操作。当这些操作结束后，它接着再取下一条指令。通常情况下，这个过程是连续不断、循环往复的。

  2. 处理器的结构

    处理器主要由三个部分组成：

  • 寄存器：在处理器内部用来临时存储计算数据。
  • 算术逻辑单元（ALU）：ALU是专门负责运算的电路，可以计算加法、减法或者乘法，也可以做逻辑运算。
  • 控制器：在指令执行过程中，负责给各个部件发送控制信号，使各个部件在某个正确的时间点上执行某个动作；同时，它还负责决定在某个时间点上哪个部件有权使用总线，以避免发生冲突。

寄存器

  1. 寄存器有8位、16位、32位、64位的，多少位寄存器就可以存放多少比特。

  2. INTEL8086是一个16位处理器，它包含16位的寄存器和16位的算术逻辑单元。

  3. 比特、字节、字、双字之间的换算关系：

    1字节（Byte）= 8比特（bits）

    1字（word）= 2字节

    1双字（double world, dw）= 2字 = 4字节

内存

  1. 内存是计算机内部最主要的存储器，通常只和处理器相连，所以叫做内存储器或者主存储器，简称内存或主存。

  2. 内存按字节来组织，单次访问的最小单位是1字节，内存中的每个字节都对应着一个地址。

  3. 处理器如何访问内存？
    为了访问内存，处理器需要给出一个地址（访问那一块内存）。访问包括读和写，为此，处理器还要指明，本次访问是读访问还是写访问（如何访问）。如果是写访问，则还要给出待写入的数据。

  4. 低端字节序：低字节存放在低地址，高字节存放在高地址（低低高高）。

指令

  1. 指令：处理器的设计者用某些数来指示处理器做进行的操作，称为指令（Instruction），或者叫机器指令，因为只有处理器才认得它。比如，指令F4H表示让处理器停机。

  2. 指令和数据对计算机来说都是一串二进制的0和1（一组高低电平组合），没有区别。

  3. 指令的执行：指令是集中存放在内存中的，一条接着一条，处理器的工作是自动按顺序取出并加以执行。

  4. 为什么要分数据段和代码段？

    因为处理器是自动按照顺序取指令并加以执行的，如果在指令中混杂了非指令的数据会导致处理器无法识别从而不能正常工作。为此，指令和数据要分开存放，分别位于内存中的不同区域，存放指令的区域叫代码段，存放数据的区域叫数据段，数据段和代码段不一定要挨在一起。

    在运行程序之前，需要为程序随机分配一块空闲的内存空间，然后将数据段的起始位置和代码段的起始位置告诉处理器，这样处理器就知道该程序的指令和数据存放在哪里了。

  5. 指令由操作码和操作数（可省略）组成。

  • 操作码决定了如何执行该指令、该指令是要做什么。
  • 操作数是指令操作的对象。如果这个数已经在指令中给出了，不需要再次访问内存，那这个数就是立即数；相反，如果指令中给出的是地址，真正的数还需要用这个地址访问内存才能得到，那么它就不是立即数。

内存分段机制

  1. 逻辑地址（相对地址）：当采用分段策略之后，一个内存单元的地址就可以用逻辑地址——“段地址：偏移地址”表示。

  2. 为了在硬件一级提供对“段地址：偏移地址”内存访问模式的支持，处理器至少要提供两个段寄存器，分别是代码段寄存器（Code Segment, CS）和数据段寄存器（Data Segment, DS）。在开始访问之前，要首先设置好CS和DS的内容。

  3. 指令中给出的是操作数的段内偏移地址。当处理器需要访问内存中的某个数据时，就把DS中的数据段起始地址和指令中提供的段内偏移相加，来得到访问内存所需的物理地址。

  4. 为什么要使用逻辑地址（相对地址）而不是物理地址（绝对地址）？

    内存中的每一个字节都有一个独一无二的物理地址，处理器访问内存最终都是使用物理地址来访问的，即使是逻辑地址也要通过计算得到物理地址才可以访问内存。

    使用逻辑地址的原因简而言之就是为了满足程序重定向的需求。

    如果要在程序中使用物理地址（绝对地址，固定不变的地址），就需要知道每一次运行时数据和代码的在内存中的确切位置然后写死。但是程序每一次运行时被分配到的内存空间是不确定的，数据段和代码段的位置也就不确定，进而导致数据和代码的位置都是不确定的，不可能提前知道，也不可能保持不变。因此，为了保证程序的正常运行，我们在代码中应该使用相对于段地址（段起始位置的内存地址）的段内偏移地址，只要程序写好了段内偏移地址就不会发生改变了。

8086的寄存器们

  1. 8086有16个通用寄存器，分别是AX, BX, CX, DX, SI, DI, BP, SP。其中，前四个寄存器中的每一个，都可以分成两个8位的寄存器来使用，分别是AH, AL, BH, BL, CH, CL, DH, DL。
  2. 8086内部有4个段寄存器。其中：

  • CS是代码段寄存器

  • DS是数据段寄存器

  • ES是附加段寄存器（Extra Segment Register）。

    附加段的含义是，它是额外赠送的礼物。当需要在程序中同时使用两个数据段时，DS指向一个，ES指向另一个。可以在指令中指定使用DS和ES中的哪一个，默认为DS。

  • SS是段寄存器

  1. IP是指令指针（Instruction Pointer）寄存器。IP只和CS一起使用，而且只有处理器才能改变它的内容。当一段代码开始执行时，CS只想代码段的起始地址，IP指向段内偏移。这样，由CS和IP共同形成逻辑地址，并由总线接口部件变换成物理地址来取指令。然后，处理器会自动根据当前指令的长度来改变IP的值，使它指向下一条指令。
  2. 8086访问数据的方式为：把DS中的数据段起始地址和指令中提供的段内偏移相加，来得到访问内存所需的物理地址。

8086的地址

  1. 8086的物理地址是20位，最多可访问$2^{20}=1$MB的内存，地址范围为00000H到FFFFFH。

  2. 8086的段地址和段内偏移地址都是16位。

  3. 如何根据“段地址：偏移地址”计算物理地址？

    段地址左移4位 + 偏移地址 = 物理地址

    举例：逻辑地址F000H:052DH对应的物理地址为F0000H+052DH=0F52DH

  4. 由于段地址是16位的，因此最多有$2^{16}=65536$个段，段地址依次为0000H、0001H、…、FFFFH。

  5. 由于偏移地址是16位的，因此每个段最多有$2^{16}=65536$个字节（64KB）。

  6. 段起始位置的物理地址由段地址左移4位得到，因此段起始位置的物理地址最后4个比特位一定是0，以XXXX0H的形式出现。或者说段起始位置的物理地址一定是16的倍数，这称为是16字节对齐的。

汇编语言编译器NASM

  1. 指令本质上是一个二进制01串，为了克服机器指令难以书写和理解的缺点，人们用一些容易理解和记忆的符号，也就是助记符，来表示指令，这就产生了汇编语言。汇编语言的本质：机器语言的助记符。
  2. 按照惯例，汇编语言源程序文件的扩展名为.asm，不过你也可以使用其他扩展名。
  3. 对汇编语言源程序文件进行编译（汇编程序的编译过程也称为“汇编”），就可以将mov、add等这些助记符号组成的汇编语言源程序转换成相对应的机器指令01串，编译生成的文件称为二进制文件或者可执行文件，比如windows系统中的.exe文件。

使用NASM编译汇编程序

  1. 基本格式：nasm -f <format> <filename> [-o <output>]

    该命令将对汇编语言源程序文件<filename>进行编译，生成文件的文件格式为<format>、文件名为<output>。

  2. 常用编译命令：

  • -f：指定编译生成的文件格式

    • windows系统用-f win32或者-f win64
    • linux系统用-f elf
    • 裸机用-f bin，此时生成的是纯二进制文件，也就是不归属于任何操作系统的文件系统的文件

  • -o：指定编译后的文件的名称 否则会有默认的文件名称

  1. 举例：
  • nasm -f elf myfile.asm -o myfile.o
  • nasm -f bin exam.asm -o exam.bin
